# Анализ продаж и складских запасов (на основе неструктурированных Excel-данных)

## 1. Обзор проекта

Проект направлен на преобразование неструктурированных Excel-отчетов (выгрузок из ERP-системы) в аналитически пригодный формат с последующим анализом продаж и складских остатков.

Основная цель — построение надежной аналитической базы для принятия решений в области управления запасами, закупок и оценки эффективности поставщиков.

---

## 2. Бизнес-контекст и проблема

В реальной практике компании часто сталкиваются с тем, что:

- Данные выгружаются из ERP в неудобном формате
- Excel-файлы содержат служебную информацию (заголовки, текстовые блоки)
- Отсутствует единая структура данных
- Невозможно напрямую проводить анализ без предварительной обработки

В данном проекте исходные данные представляли собой типичный пример «грязных» бизнес-данных, непригодных для аналитики без трансформации.

---

## 3. Ключевые проблемы данных

Анализ исходных файлов выявил следующие критические проблемы:

- Отсутствие корректных заголовков колонок (`Unnamed: ...`)
- Наличие текстовых блоков (например, «Отчет о поступлении товаров»), не относящихся к данным
- Заголовки таблиц встроены в строки данных
- Смешанные типы данных (текст в числовых колонках)
- Различные форматы данных в рамках одной колонки
- Нестандартизированные наименования поставщиков (включая город, например: «г. Москва»)
- Частично заполненные или пустые строки

---

## 4. Подход к обработке данных (Data Engineering Layer)

Для решения указанных проблем был реализован многоэтапный pipeline обработки данных:

### 4.1 Восстановление структуры таблицы
- Определение строки с реальными заголовками
- Переназначение колонок вручную
- Удаление служебных строк и метаданных

### 4.2 Очистка данных
- Преобразование числовых значений через `pd.to_numeric()`
- Приведение дат к стандартному формату (`pd.to_datetime`)
- Удаление некорректных записей

### 4.3 Нормализация
- Очистка названий поставщиков (удаление города, лишних символов)
- Стандартизация категориальных данных
- Унификация форматов

### 4.4 Формирование аналитической модели
- Создание чистых таблиц:
  - продажи (sales)
  - остатки (stock)
- Подготовка данных для агрегации и анализа

---

## 5. Методология анализа

После подготовки данных была реализована аналитическая модель:

### Агрегация:
- Выручка по поставщикам
- Продажи по товарам
- Динамика продаж по датам

### Метрики:
- Общая выручка
- Объем продаж
- Количество транзакций
- Уникальные поставщики

---

## 6. Ключевые инсайты

### Концентрация выручки
- Основная доля выручки формируется ограниченным числом поставщиков

### Неэффективные запасы
- Обнаружены позиции с низкой оборачиваемостью

### Структура продаж
- Часть товаров генерирует непропорционально высокий вклад в выручку

### Качество данных
- Без этапа очистки проведение анализа было бы невозможно или привело бы к искаженным результатам
